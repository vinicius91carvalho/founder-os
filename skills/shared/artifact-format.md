# Artifact Output Format

All artifacts generated by FounderOS follow this standard format. This ensures consistency, traceability, and professional presentation.

---

## Output Path Convention

All generated artifacts MUST be saved to:

```
artifacts/<company-name>/<area>/<timestamp>-<filename>.<extension>
```

| Component | Format | Example |
|---|---|---|
| `company-name` | kebab-case | `reconcile-ai` |
| `area` | kebab-case category | `business-plan`, `pitch-deck`, `validation` |
| `timestamp` | `YYYY-MM-DD_HHmm` | `2026-03-22_1430` |
| `filename` | kebab-case descriptor | `business-plan`, `pitch-deck` |
| `extension` | file type | `.md` |

**Examples:**
```
artifacts/reconcile-ai/business-plan/2026-03-22_1430-business-plan.md
artifacts/reconcile-ai/pitch-deck/2026-03-22_1530-pitch-deck.md
artifacts/reconcile-ai/validation/2026-03-22_1200-validation-scorecard.md
```

See `artifacts/README.md` at the project root for the full list of areas.

---

## Standard Artifact Structure

```markdown
---
artifact: [artifact type — see Artifact Types below]
generated_by: [comma-separated list of skill names that contributed]
startup: [startup name or idea description]
date: [YYYY-MM-DD generation date]
version: [1.0, incremented on regeneration]
research_sources: [list of URLs if web research was used]
---

# [Artifact Title]

> [One-sentence summary of what this artifact contains and its purpose]

---

[Content organized by sections appropriate to the artifact type]

---

## Assumptions & Limitations

- [List key assumptions made during generation]
- [Note any data gaps or areas requiring verification]
- [Flag anything that requires professional review]

## Disclaimer

[Appropriate disclaimer from shared/disclaimer-templates.md]

## Next Steps

- [ ] [Recommended action 1]
- [ ] [Recommended action 2]
- [ ] [Recommended action 3]

## Related Artifacts

- [Link to related artifact 1 — e.g., "See Business Plan for detailed financials"]
- [Link to related artifact 2]
```

## Artifact Types

### 1. Business Plan

- **Artifact type:** `business-plan`
- **Generated by:** Orchestrator (synthesizing all 12 skills)
- **Sections:** 13 parts (Investor One-Pager through Appendices)
- **Disclaimer:** General + Financial + Legal
- **Size target:** 15,000-25,000 words (comprehensive)

### 2. Pitch Deck

- **Artifact type:** `pitch-deck`
- **Generated by:** Orchestrator (primarily skills 01-03, 05, 08-10)
- **Sections:** 10 slides following Sequoia/YC hybrid format
- **Disclaimer:** General + Financial
- **Size target:** 2,000-4,000 words (concise, presentation-ready)

## Formatting Guidelines

### Headers
- Use `#` for artifact title only
- Use `##` for major sections
- Use `###` for subsections
- Use `####` sparingly for detail sections

### Data Presentation
- Use tables for comparisons, metrics, timelines
- Use bullet lists for action items and recommendations
- Use numbered lists for sequential steps
- Use blockquotes for key insights or callouts

### Financial Data
- Always include currency (USD unless specified)
- Use consistent number formatting (e.g., $1.2M, not $1,200,000)
- Label all assumptions explicitly
- Provide ranges for projections, not single numbers

### Research Data
- Cite sources inline: "According to [Source] (URL), the market is..."
- Date all market data: "As of 2026, the TAM is..."
- Distinguish between hard data (cited) and estimates (labeled)

### Cross-References
- Reference other artifacts: "See [Artifact Name] for detailed analysis"
- Reference specific sections: "See Business Plan Part IV: Market Analysis"
- Reference source skills: "Generated from Skill 02: Business Model analysis"
