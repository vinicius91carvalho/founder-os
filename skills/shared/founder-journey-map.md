# Founder Journey Map

The FounderOS organizes the founder journey into 5 stages. Each stage maps to specific skills, has entry/exit criteria, and produces outputs that feed later stages.

---

## Stage 1: Ideation

**Goal:** Validate whether this idea is worth pursuing.

**Skills:**
- **01 — Idea Validation** (primary): Validate problem, market, founder-market fit
- **02 — Business Model** (secondary): Explore revenue model viability

**Entry Criteria:**
- Founder has a startup idea (can be raw or partially formed)
- Founder can articulate the problem they want to solve

**Exit Criteria:**
- GO/PIVOT/KILL verdict from Idea Validation
- If GO: initial business model hypothesis selected
- If PIVOT: revised hypothesis ready for re-validation
- If KILL: documented reasons, founder moves to next idea

**Key Frameworks:**
- Kevin Hale/YC Validation Framework
- Tom Bilyeu 60-Minute Validation
- Mom Test (Rob Fitzpatrick)
- April Dunford Competitive Positioning (early exploration)

**Outputs for Later Stages:**
- Validation evidence (market signals, customer interviews, competitive gaps)
- Founder-market fit assessment
- Initial business model hypothesis
- Market size indicators

---

## Stage 2: Pre-Launch

**Goal:** Prepare everything needed to go to market — product definition, funding strategy, and launch plan.

**Skills:**
- **03 — Fundraising**: Prepare materials, identify investors, understand terms
- **04 — Go-to-Market**: Plan launch strategy, waitlist, first customers
- **05 — Product**: Define PRD, roadmap, user stories

**Entry Criteria:**
- Idea validated (GO verdict from Stage 1)
- Business model hypothesis selected
- Founder ready to commit resources

**Exit Criteria:**
- Product PRD complete with prioritized roadmap
- Funding strategy decided (bootstrap vs. raise, target amount)
- If raising: pitch deck complete, investor list built
- GTM plan ready with pre-launch tactics defined
- First 100 customers playbook prepared

**Key Frameworks:**
- Sequoia/YC Hybrid Pitch Deck (10 slides)
- SAFE/Convertible Note Mechanics
- Kevin Yien/Square PRD Template
- RICE Scoring, INVEST Criteria
- Lenny Rachitsky B2C/B2B Growth Research
- Racecar Growth Framework

**Outputs for Later Stages:**
- Pitch deck and fundraising materials
- Product roadmap with RICE scores
- GTM launch playbook with timeline
- Funding terms and cap table projections

---

## Stage 3: Launch

**Goal:** Execute the go-to-market plan, acquire first customers, and establish initial revenue.

**Skills:**
- **06 — Sales**: Execute founder-led sales, build pipeline
- **07 — Marketing & Brand**: Establish brand, launch content strategy, build distribution

**Entry Criteria:**
- Product built (or MVP ready)
- GTM plan prepared (from Stage 2)
- Initial positioning and messaging defined

**Exit Criteria:**
- First 100 customers acquired (or significant traction signal)
- Sales methodology selected and initial pipeline built
- Brand voice established and content strategy executing
- Revenue or clear path to revenue demonstrated

**Key Frameworks:**
- MEDDIC (enterprise), BANT (SMB), Challenger Sale (differentiated products)
- 80/20 Distribution Principle (distribute > create)
- Pillar Content Strategy
- Community Building Ladder (lurker → contributor → advocate → ambassador)

**Outputs for Later Stages:**
- Sales pipeline data and conversion rates
- Brand guidelines and content library
- Customer acquisition cost by channel
- Initial customer feedback and NPS

---

## Stage 4: Growth

**Goal:** Measure what matters, optimize growth loops, and build the team and processes to scale.

**Skills:**
- **08 — Growth & Analytics**: Define metrics, run experiments, analyze cohorts
- **09 — Operations**: Hire team, set OKRs, establish governance

**Entry Criteria:**
- Product in market with paying customers
- Initial traction data available
- Need to scale beyond founder-led everything

**Exit Criteria:**
- North Star metric defined and tracked
- Growth experiments running with statistical rigor
- Key hires made with structured interview process
- OKR framework adopted
- Board reporting cadence established (if applicable)

**Key Frameworks:**
- AARRR Pirate Metrics (Acquisition, Activation, Retention, Revenue, Referral)
- North Star Metric Selection
- A/B Testing with Sample Size Calculations
- Liz Wessel/Google OKR Framework
- Structured Interview Scorecards

**Outputs for Later Stages:**
- Growth metrics dashboard
- Cohort retention analysis
- Organizational structure and OKRs
- Board deck and investor update templates

---

## Stage 5: Scale

**Goal:** Build the operational backbone — finance, customer success, and legal/compliance — to support sustainable growth.

**Skills:**
- **10 — Finance & Accounting**: Manage burn, forecast cash, prepare for next round
- **11 — Customer Success**: Reduce churn, expand accounts, build CS function
- **12 — Legal & Compliance**: Structure entity, manage cap table, ensure compliance

**Entry Criteria:**
- Revenue growing, team growing
- Financial complexity increasing (payroll, multiple revenue streams, investor reporting)
- Customer base large enough to need systematic CS
- Legal/compliance obligations increasing

**Exit Criteria:**
- Monthly close process running
- Runway scenarios modeled (base/optimistic/pessimistic)
- Customer health scoring and churn prevention active
- Entity properly structured, cap table clean, compliance requirements met
- Ready for next funding round (if applicable)

**Key Frameworks:**
- Burn Rate Analysis (gross vs. net)
- 13-Week Cash Flow Forecast
- Time-to-First-Value (TTFV) for customer success
- Churn Segmentation by Timing (30-day vs. 6-month)
- Entity Selection (LLC/C-Corp/S-Corp)
- Standard Vesting (4-year cliff)
- SAFE Mechanics (pre/post-money)

**Outputs:**
- Financial model with 3-year projections
- Runway calculator with scenarios
- Customer health dashboard
- Clean cap table with dilution modeling
- Compliance checklist by jurisdiction

## Routing Decision Tree

```
START: "Tell me about your startup."
│
├─ No idea yet / raw idea
│   └─ → Stage 1: Start with Skill 01 (Idea Validation)
│
├─ Idea validated, building/planning
│   ├─ Need funding? → Skill 03 (Fundraising) first
│   ├─ Need product definition? → Skill 05 (Product) first
│   └─ Ready to launch? → Skill 04 (GTM) first
│   └─ → Stage 2: Run relevant Pre-Launch skills
│
├─ Product in market, acquiring customers
│   ├─ No sales process? → Skill 06 (Sales)
│   ├─ No brand/marketing? → Skill 07 (Marketing)
│   └─ → Stage 3: Run relevant Launch skills
│
├─ Growing but need systems
│   ├─ No metrics/analytics? → Skill 08 (Growth)
│   ├─ Hiring/scaling team? → Skill 09 (Operations)
│   └─ → Stage 4: Run relevant Growth skills
│
├─ Scaling, need operational backbone
│   ├─ Financial complexity? → Skill 10 (Finance)
│   ├─ Churn/retention issues? → Skill 11 (Customer Success)
│   ├─ Legal/compliance needs? → Skill 12 (Legal)
│   └─ → Stage 5: Run relevant Scale skills
│
└─ Wants full system run (Business Plan / Pitch Deck)
    └─ → Run Orchestrator end-to-end: Stage 1 → 5, generate artifacts
```

## Partial Run Support

Not every founder needs all 12 skills. Common partial run patterns:

| Founder Situation | Skills to Run | Skip |
|-------------------|---------------|------|
| "I just have an idea" | 01, 02 | 03-12 |
| "Validated idea, need to raise" | 02, 03, 05 | 01, 04, 06-12 |
| "Building product, planning launch" | 04, 05, 07 | 01-03, 06, 08-12 |
| "Launched, need growth" | 06, 07, 08 | 01-05, 09-12 |
| "Growing, need to professionalize" | 09, 10, 11, 12 | 01-08 |
| "Need a complete Business Plan" | All 12 | None |
| "Need a Pitch Deck only" | 01, 02, 03, 08 | 04-07, 09-12 |

## Cross-Stage Data Flow

Each stage's outputs become the next stage's inputs. The orchestrator maintains a **journey context** that accumulates:

```
Journey Context = {
  validation_verdict: from Skill 01,
  business_model: from Skill 02,
  funding_strategy: from Skill 03,
  gtm_plan: from Skill 04,
  product_prd: from Skill 05,
  sales_pipeline: from Skill 06,
  brand_voice: from Skill 07,
  growth_metrics: from Skill 08,
  org_structure: from Skill 09,
  financial_model: from Skill 10,
  cs_strategy: from Skill 11,
  legal_structure: from Skill 12
}
```

This context is passed to artifact generators (Business Plan, Pitch Deck) to produce integrated, consistent documents.
